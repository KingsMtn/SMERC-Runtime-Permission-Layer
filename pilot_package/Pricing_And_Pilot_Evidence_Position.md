# SMERC Pricing And Pilot Evidence Position

## Purpose

This document explains how SMERC should price early pilots and how to position the Decision Lifecycle Ledger without overstating it.

The goal is to sell a measurable pilot outcome, not a vague framework and not a premature production security platform.

## Core Commercial Promise

SMERC should be sold around this question:

> In 30 to 90 days, can recoverability-aware scoring change security or platform reviewer judgment in a useful, measurable way before AI-agent or automation actions create side effects?

The Decision Lifecycle Ledger supports that promise by preserving the evidence trail. It should not be sold as a standalone compliance guarantee.

## Recommended Early Pricing

| Offer | Price | Best fit | Required output |
| --- | ---: | --- | --- |
| CISO Technical Review | Free to `$2,500` | Serious buyer, design partner, technical reviewer, or integration partner. | Fit assessment and scoped pilot recommendation. |
| 30-Day Shadow-Mode Pilot | `$7,500-$15,000` | One workflow with existing review process and clear metadata boundary. | Decision summary, reviewer labels, artifact summary, and final report. |
| 90-Day Design Partner Pilot | `$25,000-$50,000` | Multiple workflows or one strategically important workflow with weekly review. | Decision Lifecycle Ledger evidence package, calibration notes, go/no-go recommendation, and executive findings. |
| Enterprise Expansion | Custom | Organization that has completed a pilot and wants broader deployment planning. | Scope-specific API, integration, security, retention, and support plan. |

## Why Not Price Too Low

Very low pricing can create the wrong signal for a CISO. It may make SMERC look like a hobby project instead of a serious security and governance pilot.

The pilot requires real customer time:

- security owner review
- platform owner review
- metadata mapping
- weekly decision review
- reviewer labeling
- final findings discussion

The price should reflect that this is a structured evaluation, not a simple demo.

## Why Not Price Too High Yet

SMERC should not price like a mature enterprise platform before it has:

- live pilot evidence
- buyer references
- production security review
- managed enterprise deployment path
- customer-calibrated thresholds
- durable storage and retention plan
- security/compliance documentation beyond pilot grade

The first paid pilots are evidence-building pilots, not enterprise-wide production deployments.

## Ledger Value

The Decision Lifecycle Ledger is commercially interesting because it records:

- request context
- available evidence
- SMERC evaluation
- human interaction
- execution facts
- rollback facts
- delayed outcomes
- learning recommendations

This can help security and platform teams answer:

- Did reviewers agree with SMERC?
- Did overrides improve or worsen outcomes?
- Which controls were useful?
- Which actions were hard to recover?
- Which workflows created the most irreversible exposure?
- What policy changes are justified by evidence?

## Ledger Boundary

The ledger is not enough by itself.

It does not prove value unless it contains customer pilot data. It should distinguish:

- synthetic examples
- decision-time evidence
- reviewer labels
- execution evidence
- delayed outcome evidence
- learning recommendations

SMERC should avoid implying that a ledger record is a legal compliance archive, immutable regulatory record, or proof of incident reduction without customer-approved infrastructure and evidence.

## Recommended Sales Language

Use:

> SMERC creates replayable pilot evidence so your security and platform teams can decide whether recoverability scoring improves governance decisions.

Avoid:

> SMERC proves AI-agent safety.

Avoid:

> SMERC replaces policy engines, approval systems, or compliance tooling.

Avoid:

> SMERC's ledger is a complete legal or regulatory audit record.

## Default Offer

The default offer should be:

> 30-day GitHub Actions shadow-mode pilot, `$7,500-$15,000`, one workflow, metadata-only, observe mode, weekly review, final evidence report.

For a larger organization with real urgency:

> 90-day design partner pilot, `$25,000-$50,000`, multiple workflows, Decision Lifecycle Ledger evidence package, calibration support, executive findings.
